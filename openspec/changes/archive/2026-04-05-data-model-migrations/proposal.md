## Why

The Go rewrite has no data access layer yet. All 68 PostgreSQL tables need type-safe repository functions so every domain change can read and write the database. This change delivers all `db/queries/` SQL files and confirms the repository patterns (history tables, encrypted columns, soft-delete, enum lookup tables) that production code must follow.

## What Changes

- 14 query files added under `db/queries/` covering all 9 domains: notifications, services, api_keys, templates, jobs, billing, users, organisations, inbound_sms, providers, complaints, reports, annual_limits, template_categories
- `internal/repository/` packages generated by `sqlc generate` from the query files and the seed migration from `go-project-setup`
- Explicit `InsertXxxHistory(ctx, h)` calls documented and tested for all 6 versioned entities: api_keys, provider_details, service_callback_api, service_inbound_api, services, templates
- Encrypted column handling pattern documented: service layer calls `pkg/crypto.Decrypt`/`Encrypt` explicitly on read/write for all 8 encrypted columns
- Soft-delete conventions documented for api_keys, jobs, service_email_reply_to, service_letter_contacts, service_sms_senders, templates

## Capabilities

### New Capabilities

- `repository-layer`: The sqlc-generated repository function catalogue for all domains, plus the history-write, encrypted-column, and soft-delete patterns that all domain service layers must follow

### Modified Capabilities

## Non-goals

- Domain business logic (validation, rate limiting, error translation) — handled in individual domain changes
- Connection pool configuration — handled in `go-project-setup`
- Redis caching layer over repository calls — handled in domain changes that need it
- Future schema migrations beyond the seed (contributed by each domain change as needed)

## Impact

- Requires `go-project-setup` to be complete (seed migration and sqlc.yaml must exist)
- All 18 subsequent domain changes depend on the repository packages generated here
- Running `sqlc generate` after adding queries is a required CI step from this point forward
